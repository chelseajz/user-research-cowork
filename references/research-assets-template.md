# Research Assets Template

Use these assets to keep the report reusable after delivery.

## 1. Codebook

Use when the project includes meaningful qualitative analysis.

Suggested fields:
- code name
- definition
- inclusion rule
- exclusion rule
- example quote

## 2. Segment card

Use when patterns cluster by behavior, need state, or decision logic.

Suggested fields:
- segment name
- defining behavior or need
- typical trigger or scenario
- common barrier
- representative quote
- design or product implication

## 2.5 Participant profile / casebook entry

Use when interview work needs richer qualitative grounding.

Suggested fields:
- participant id
- decision-relevant profile
- linked survey attributes if available
- core situation
- main goal
- main barrier or tension
- notable quote(s)
- why this case matters
- linked finding(s)
- selected reason / typical reason

If a case is described as typical or representative, the selected reason must match both the segment label and the direct quote evidence. If they conflict, relabel the case or move it to a more accurate bucket.

If the casebook is intended to represent the study, also preserve a short case sampling note with:
- sampling logic
- coverage dimensions
- overrepresented and underrepresented profiles

## 3. Quote bank

Use to preserve vivid source material without overcrowding the main body.

Suggested fields:
- quote id
- respondent id
- profile
- topic
- scene
- quote text
- linked finding(s)

For rigorous work, also include:
- source question
- need state or adoption state
- segment or label

## 4. Decision log

Use when the report should support downstream action.

Suggested fields:
- decision area
- research input used
- recommendation
- owner
- next step
- metric to watch

## 5. Key chart index

Use when the package includes many charts and business readers need a fast-entry layer.

Suggested fields:
- chart id
- human-readable title
- business question answered
- where used in the main report

## 6. Decision and conversion analysis

Use when the study involves adoption, borrowing, purchase, trust, or other stepwise decisions.

Suggested fields:
- decision stage
- driver
- blocker
- strongest evidence
- implication
- supporting quote(s)

## 7. Product feedback deep-dive

Use when users react to a concept, prototype, demo, or proposition.

Suggested fields:
- feature or idea
- positive signal
- hesitation or misunderstanding
- feasibility dependency or open question
- implication for product or communication
- supporting quote(s)

For decision-heavy or product-feedback-heavy work, the companion asset should not be statistical-only. Preserve enough original voice that the trade-off, tension, or misunderstanding is audible in the user's own words.

## Minimal packaging rule

For substantial studies, at least preserve:
- one finding ledger
- one quote bank
- one decision log

For thick mixed-method work, usually also preserve:
- one chartbook
- one key chart index
- one casebook
- one case sampling note
